# UX 감사 — 전문가 페르소나

- **감사일**: 2026-05-09
- **페르소나**: 부동산 경매 10년차 베테랑 투자자 + 공인중개사 자격증 보유자
- **방법**: 코드/문서 정적 분석 (코드 변경 없음)
- **범위**: 전체 앱 — services, models, components, views, seeds, prompts, adapters

---

## 페르소나 정의

- 연 30~50건 입찰 경험. 낙찰 받은 물건 100건 이상
- 권리분석, 배당표 작성, 명도, 협상, 강제집행 모두 직접 수행
- 대법원 경매정보 사이트(courtauction.go.kr)를 매일 사용
- 등기부등본, 건축물대장, 토지이용계획확인서 자유자재로 해석
- 도구는 효율성과 정확성으로 평가. 전문가용 데이터/기능이 부실하면 즉시 신뢰 잃음
- 시간이 금이라 클릭 한 번 줄여주는 도구를 선호
- 이미 자기만의 엑셀/노션 워크플로우가 있음 — 이걸 대체할 가치가 있어야 씀

## 전문가 종합 소견

### 출시 전 반드시 막아야 할 Critical (사용자 금전 손실 직결)

1. **인수금액 계산의 단순 합산 문제 (#2, #3)** — 채권액 합산이 실제 인수금액과 이중계상되거나 배당 후 금액과 다르다. 사용자가 이 도구의 "총 위험금액"을 보고 입찰 안 했는데 사실 안전한 매물이거나, 반대로 안전이라 봤는데 인수해야 했던 케이스가 모두 가능. 출시 전 면책 강화 + UI에 "추정치이며 베테랑 검증 필요" 강한 경고 필수.
2. **PDF 분석 case_number를 LLM이 추출 → 타인 매물 데이터 덮어쓰기 가능 (#7)** — 가장 위험한 데이터 무결성 결함. 사용자가 사건번호를 먼저 입력하지 않고 PDF만 던지는 경로를 막아야 함.
3. **약관/개인정보처리방침 placeholder 상태 (#36)** — 5/19 출시 시 회원가입 받으면 즉시 법적 risk.
4. **dividend_requested 필드 mismatch (#8)** — 코드와 prompt가 어긋나 항상 "배당요구 ✗" 표시. 사용자가 잘못된 정보 기반으로 명도 시뮬 진입.
5. **공유자우선매수권/공유지분 매각 처리 부재 (#4, #5)** — 베테랑이 5분 만에 발견할 핵심 누락. priority "중" 분류는 전문가 신뢰 즉시 상실.
6. **rights_validator의 동일자 전입 처리 (#1)** — 익일 0시 효력 발생 원칙 미반영. 가장 자주 발생하는 케이스.
7. **소수 입력으로도 "안전" 판정 가능 (#13, #14)** — 입력 1개로 safe 결과 가능. 데이터 누락 안전망 강화 필수.
8. **bid_opinion의 권유성 문구 (#38)** — 변호사법/중개사법 risk + 사용자 손해 시 책임 분쟁.

### 출시 후 추가해야 할 전문가 기능 Top 5

1. **다물건 비교 보드 + Excel/CSV import/export (#20, #23, #30, #40)** — 베테랑이 매일 쓰는 워크플로우. 이게 없으면 도구를 5건 이상 분석에 쓸 동기 없음.
2. **배당표 시뮬레이터 + 소액임차 최우선변제 자동 계산 (#3, #27)** — 권리분석의 본질. 정확한 인수금액 산출의 핵심.
3. **매각기일/잔금납부/인도명령 6개월 마감 등 D-day 알림 (#28, #33, #34, #35)** — auction_schedule 모델은 있는데 사용 안 함. 가장 적은 노력으로 가장 큰 가치.
4. **물건별 메모/사진/임장노트 (#21)** — UserProperty에 free-form 필드 추가. 베테랑이 도구에 lock-in되는 핵심 기능.
5. **입찰가 산정 보조: 회차별 저감률 + 인근 낙찰가 통계 + DSR 한도 (#24, #29)** — 베테랑이 도구를 "분석"이 아닌 "결정"에 쓰게 만드는 기능. 단순한 수익률 계산을 넘어 "얼마 써야 하나"의 답을 줌.

### 종합 평가

**5/19 단일서버 출시는 가능하지만, 베테랑(연 30~50건 입찰자) 페르소나에 대한 retention은 1주를 못 넘긴다.** 도구가 (a) 정확성에서 검증 못 받고(인수금액·세금·배당 자동계산이 단순화된 추정), (b) 워크플로우 면에서 단건 처리에만 최적화(다건 비교/메모/import 부재), (c) 베테랑 필수 항목(공유자우선매수, 공동지분, 매각기일 알림)을 빠뜨렸다.

**현재 강점:** 89개 체크리스트 시드 품질, dependency 트리 모델, AI evidence 노출, registry timeline 시각화, 명도 시뮬레이터의 단계 분기 — 이 5가지는 입문자~중급자에겐 우수.

**현재 약점:** 베테랑 입장에선 "내가 이미 엑셀로 더 정확하게 한다"는 결론. 베테랑 retention이 목표라면 출시 후 **고급 데이터(낙찰가 통계, 시행령 별표, 배당 시뮬)** 와 **워크플로우 도구(비교/메모/export)** 두 축으로 동시 보강해야 한다.
