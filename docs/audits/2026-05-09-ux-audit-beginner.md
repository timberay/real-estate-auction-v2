# UX 감사 — 초심자 페르소나

- **감사일**: 2026-05-09
- **페르소나**: 30대 직장인, 부동산 경매 완전 초심자
- **방법**: 코드/문서 정적 분석 (코드 변경 없음)
- **범위**: 전체 앱 모든 화면

---

## 페르소나 정의

- 부동산 경매를 한 번도 해본 적 없음. "경매" 하면 옥션/이베이 떠올림
- 친구가 "법원 경매로 시세보다 싸게 집을 살 수 있다"고 해서 호기심으로 가입
- 모르는 용어: 권리분석, 명도, 배당, 말소기준권리, 유찰, 임차인 대항력, 최저매각가, 감정가, 매각기일, HUG, 사건번호 등 — **거의 모든 경매 용어**
- 컴퓨터는 일반적인 수준. 모바일도 자주 씀
- 인내심 부족. 이해 안 되면 바로 이탈
- 한국어만 사용

---

## 첫 인상 (서술형)

루트(`/`)에 들어왔는데 곧바로 온보딩으로 튕겨버려서 "이 사이트가 뭐 하는 곳인지" 한 번도 못 봤습니다(`home_controller.rb:5-9`). "법원 경매로 시세보다 싸게 산다"는 친구 말 듣고 들어왔는데, 헤더에 그냥 "Real Estate Auction"이라는 영문 브랜드명만 있고(`app/components/header/component.html.erb:6`) 회사 소개·기능 설명·"무엇을 도와주는 도구인지" 안내가 0%입니다. 이 상태에서 첫 화면이 "투자 가능한 유용자금을 입력하세요"라고 시키니, 저는 "왜 이걸 입력해야 하지?", "유용자금이 뭐지?"부터 막힙니다.

## 초심자 종합 소견

### 가장 큰 이탈 지점 3곳

1. **온보딩 첫 단계 (`step1.html.erb`)** — 사이트 정체를 모른 채 "유용자금"이라는 회계 용어부터 만남. 친구가 추천했어도 여기서 절반은 닫을 것 같아요. 랜딩 페이지 부재 + 용어 장벽이 동시에 발생하는 진입 절벽.

2. **인스펙션 4탭 화면 (`inspections/tabs/edit.html.erb` + 44개 체크리스트)** — "매각물건명세서 소멸되지 아니하는 것 비고란…" 같은 한 문장에 모르는 용어 6개. 이걸 89개(혹은 44개) 채워야 하는데 PDF 업로드 흐름이 막혀 있고(자동분석 비활성), 수동분석은 JSON 복붙이라 더 어려움. 초심자가 가장 오래 막히는 곳.

3. **명도 시뮬레이터 점유자 유형 선택 (`simulator/select_type.html.erb`)** — "후순위 임차인 (배당 수령)" 같은 카드 4개 중 하나를 골라야 시작. 모르겠다고 답할 옵션이 없고, 모든 옵션이 한자 병기 전문용어라 결정 마비. + 결과 화면의 step/branch 상세는 한 줄짜리 빈 페이지(`steps/show.html.erb`).

### 출시 전 반드시 고쳐야 할 Top 5

1. **랜딩 페이지 신설** — 비로그인 시 root에서 "이 서비스가 뭔지" 한 화면. (#1, #32)
2. **약관·개인정보 처리방침 실내용 작성** — 현재 "작성 중" 한 줄. OAuth 동의가 무효화될 위험. (#29)
3. **eviction_guide 단계/분기 상세 페이지(`steps/show`, `branches/show`) 실제 콘텐츠 채우기** — 지금 한 줄짜리 stub. 시뮬레이터 결과에서 클릭 시 빈 페이지로 가는 깨진 경험. (#21)
4. **체크리스트 항목에 인라인 용어 설명/툴팁 + "초심자 모드"** — 44개 항목의 description은 잘 쓰여 있지만 question 자체가 어렵고 question 옆에서 즉시 단어 풀이가 안 나옴. AI 자동분석이 비활성된 지금이라면 더더욱 인스펙션 화면의 친절도가 결정적. (#5, #6)
5. **외부 LLM 전송 데이터 범위 명시 + 자동분석 탭 상태 분리** — 현재 자동분석이 비활성인데 탭은 살아 있고 "ChatGPT/Claude/Gemini로 전송"만 적혀 무엇이 전송되는지·언제 폐기되는지 안 적혀 신뢰 큰 손상. (#10, #28)

### 보너스 — 사이드바 라벨 정리

"물건 목록"(시스템 검색)과 "내 물건"(내가 담은 것)의 구분이 직관적이지 않아 첫 사용자가 어떤 메뉴부터 클릭해야 할지 모릅니다(#33). 메뉴를 시간 순서로(예산 → 찾기 → 분석 → 보고서 → 명도) 시각적으로 배치하거나, 라벨을 액션 중심("물건 찾기" / "내가 분석 중")으로 바꾸면 매뉴얼 6단계 흐름과 사이드바가 일치해 학습 부담이 감소합니다.
