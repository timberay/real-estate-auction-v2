### Task 2: Sidebar — Active Menu Highlight via Prefix Matching

**Files:**
- Modify: `app/components/sidebar/component.rb:41-43` (the `active?` method)

- [ ] **Step 1: Change `active?` from exact match to prefix match**

In `app/components/sidebar/component.rb`, change the `active?` method (lines 41-43) from:

```ruby
def active?(item)
  item.path.present? && item.path == @current_path
end
```

to:

```ruby
def active?(item)
  item.path.present? && @current_path.start_with?(item.path)
end
```

This ensures that when `@current_path` is `/properties/1/inspections/tabs/sale_document`, the "물건 목록" menu item (path: `/properties`) highlights correctly.

**Why this is safe:** The sidebar menu items have paths `/onboarding` and `/properties` — neither is a prefix of the other, so there's no false-positive risk. There is no root-path (`/`) menu item.

- [ ] **Step 2: Manually verify behavior**

The sidebar component has no unit tests currently. Verify by checking the following logic mentally or in console:

- `/properties` → "물건 목록" active (exact match, still works)
- `/properties/1/inspections/tabs/sale_document` → "물건 목록" active (prefix match, now works)
- `/onboarding` → "예산 설정" active (exact match, still works)
- `/onboarding` does NOT activate "물건 목록" (different prefix, correct)
