### 🔴 F-01 · AnalysesController#manual `rescue => e` 의 원시 메시지 노출
**위치:** `app/controllers/analyses_controller.rb:67-71`

```ruby
rescue => e
  Rails.logger.error "[ManualUpload] #{e.class}: #{e.message}\n..."
  redirect_to new_analysis_path(tab: "manual"),
    alert: "분석 결과 저장 중 오류가 발생했습니다: #{e.message}"
end
```

`e.message`가 flash로 그대로 노출됨 → DB 컬럼명/내부 경로/스택 단서가 사용자 화면(레드 토스트)에 보일 수 있음. **변경:** 사용자에게는 고정 메시지("저장 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요.") 만 표시하고 `e.message`는 logger.error로만 흘리기. 필요하면 `SecureRandom.hex(4)` 같은 incident_id를 사용자에게 안내해 로그와 매칭.

### 🟠 F-08 · 존재하지 않는 리소스를 항상 인증 게이트로 보냄
**증거:** `curl http://localhost:3000/properties/999999` → 302 → `/auth/login`

**변경(권장):** 로그인 성공 후 `session[:return_to_url]`로 돌아갔을 때 `find_by` 결과가 nil이면 properties_path로 리다이렉트 + `alert: "해당 물건을 찾을 수 없습니다."` 표시 (이미 AnalysesController에 동일 패턴 존재 — `analyses_controller.rb:8`).

### 🟠 F-09 · `unread_notification_count` 가 매 페이지 렌더마다 COUNT 쿼리
**위치:** `app/components/header/component.rb:56-58`

```ruby
def unread_notification_count
  @current_user&.notifications&.unread&.count.to_i
end
```

모든 인증된 페이지 요청마다 `SELECT COUNT(*) ... WHERE read_at IS NULL`. 알림이 거의 없는 사용자라도 N+1성 비용.

**변경:**
- `User#unread_notification_count` 캐시 (60s TTL `Rails.cache`).
- 또는 `counter_cache: :unread_notifications_count` + 알림 read 시 decrement 콜백.
- N+1 측정 후 `prosopite` 등으로 베이스라인 측정 권장 (성능 인덱스 PR `972c51a` 의 후속).

### 🟠 F-10 · `capture_return_to_url` 가 인증 유저에게도 매 GET마다 세션 쓰기
**위치:** `app/controllers/application_controller.rb:74-80`

```ruby
def capture_return_to_url
  return unless request.get? || request.head?
  return if request.path_info.start_with?("/auth")
  return if request.xhr? || turbo_frame_request?
  session[:return_to_url] = request.fullpath
end
```

`current_user` 가 있어도 세션을 갱신 → 매 요청 Set-Cookie 헤더 변화 → 캐시 효율/대역폭 미세 손실 + 의도와 무관(이미 로그인됨).

**변경:** `return if current_user` 추가하거나, `require_authenticated_user` 안에서만 저장.
